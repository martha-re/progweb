Formularios

Los formularios son parte de la vida diaria. Cuando utilizamos un formulario físico en la vida real, escribimos información y se la damos a alguien para que la procese. 

Al igual que un formulario físico, un elemento HTML <form> es responsable de recopilar información para enviar a otro lugar. Cada vez que navegamos por Internet entramos en contacto con formularios y es posible que ni siquiera nos demos cuenta. Es probable que si está escribiendo en un campo de texto o proporcionando una entrada, el campo en el que está escribiendo sea parte de un <form>.

A continuación conocerá la estructura y sintaxis de un formulario y los muchos elementos que puede contener.

Funcionamiento de los formularios

Se puede pensar en Internet como una red de computadoras que envían y reciben información. Las computadoras necesitan una solicitud HTTP para saber cómo comunicarse. La solicitud HTTP indica a la computadora receptora cómo manejar la información entrante. 

El elemento <form> es una gran herramienta para recopilar información, pero aun es necesario enviar esa información a otro lugar para su procesamiento. Es necesario proporcionar el elemento <form> con la ubicación de donde va la información de <form> y qué solicitud HTTP realizar. Ejemplo:

<form action = "/ejemplo.html" method = "POST">
</form>


En el ejemplo anterior, se crea el esqueleto de un <form> que enviará información a ejemplo.html como una solicitud POST:

El atributo de acción determina a dónde se envía la información.
Al atributo de método se le asigna un verbo HTTP que se incluye en la solicitud HTTP.

Nota: los verbos HTTP como POST no necesitan estar en mayúscula para que la solicitud funcione, pero se hace por convención. 

El elemento <form> puede contener elementos internos. Por ejemplo, un encabezado para que los usuarios sepan de qué se trata este <form>, un párrafo etc.

<form action = "/ejemplo.html" method = "POST">
  <h1> Formulario de ejemplo </h1>
  <p> A continuación aprenderemos acerca de los formularios </p>
</form>
El ejemplo anterior no recopila ninguna entrada de usuario.

Captura de texto

Para crear un campo de entrada en un formulario, es necesario un elemento <input>.

El elemento <input> tiene un atributo type que determina cómo se aprecia el campo en la página web y qué tipo de datos puede aceptar.

Uno de los valores para el atributo type es "text". Al crear un elemento <input> con type = "text",  se genera un campo de texto en el que los usuarios pueden escribir.

También es importante incluir un atributo de nombre para el <input>; sin el atributo de nombre, la información de <input> no se enviará cuando se envíe el formulario.

Ejemplo:

<form action = "/ejemplo.html" method = "POST">
  <input type = "text" name = "primer-campo">
</form>

Salida:

Después de que los usuarios escriben en el elemento <input>, el valor del atributo value se convierte en lo que se escribe dentro del campo de texto. El valor del atributo value se junta con el valor del atributo de name y se envía como texto cuando se envía el formulario. Por ejemplo, si un usuario escribió "clase de programación" en el campo de texto creado por nuestro elemento <input>:

Cuando se envía el formulario, el texto: "primer-campo = clase de programación" se envía a /ejemplo.html ya que el valor del atributo name es "primer-campo" y el valor de value es "clase de programación".

También es posible asignar un valor predeterminado al atributo value para que los usuarios tengan un campo de texto precargado cuando vean por primera vez el formulario:

<form action = "/ejemplo.html" method = "POST">
  <input type = "text" name = "primer-campo" value = "valor predeterminado">
</form>


Salida:

Etiquetas

En la sección anterior, se creó un elemento <input> pero no se incluyó nada para explicar en qué se usará. Para que un usuario identifique correctamente un <input>, se utiliza el elemento <label> con el nombre apropiado.

El elemento <label> tiene una etiqueta de apertura y cierre y muestra texto escrito entre dichas etiquetas. Para asociar un <label> y un <input>, el <input> necesita un atributo id. Luego se asocia el atributo for del elemento <label> con el valor del atributo id de <input>, como sigue:

<form action = "/ejemplo.html" method = "POST">
  <label for = "respuesta_pasatiempo"> ¿Cuál es tu pasatiempo favorito? </label>
  <br>
  <input type = "text" name = "pasatiempo" id = "respuesta_pasatiempo">
</form>


Salida:

Otro beneficio de usar el elemento <label> es que cuando se hace clic en este elemento, el <input> correspondiente se resalta/selecciona.

Captura de contraseñas

Imagine todas esas veces que ha sido necesario escribir información confidencial, como una contraseña o un PIN, en un formulario. Para estos casos se utiliza el atributo type = "password" en <input>.

Un elemento <input type = "password"> reemplaza el texto ingresado con otro carácter como un asterisco (*) o un punto (•). Ejemplo:

<form>
  <label for = "clave"> Contraseña: </label>
  <input type = "password" id = "clave" name = "clave">
</form>


Salida:

Aunque el campo de la contraseña oculta el texto de la contraseña, cuando se envía el formulario, se envía el valor del texto. Esto es, si se escribe "claveSecreta" en el campo contraseña, se envía "clave = claveSecreta" junto con la otra información en el formulario.

Entradas Numéricas

Para configurar la entrada para que sea numérica, se utiliza type = "number", con esto se restringe la entrada a solo números (y algunos caracteres especiales como -, + y .). También se puede proporcionar un atributo de incremento/decremento (step), que crea flechas dentro del campo de entrada para aumentar o disminuir el valor de value de acuerdo a lo que se establezca en step. Ejemplo:

<form>
  <label for = "semestre"> Semestre cursado: </label>
  <input id = "semestre" name = "semestre" type = "number" step = "1">
</form>

Salida:

Rangos

Usar un <input type = "number"> esta bien,  si se desea permitir que los usuarios ingresen cualquier número. Pero, si es necesario limitar la entrada de los usuarios, entonces el tipo a utilizar es "range",  que crea un control deslizable.

Para establecer los valores mínimo y máximo del control deslizable, se asignan valores a los atributos min y max de <input>. También se puede controlar el funcionamiento del control deslizable asignando un valor al atributo step. Ejemplo:

<form>
  <label for = "volumen"> Control de volumen </label>
  <input id = "volumen" name = "volumen" type = "range" min = "0" max = "100" step = "1">
</form>

Salida:

En el ejemplo anterior, cada vez que el control deslizable se mueve en uno, el valor del atributo de value de <input> cambia.

Cajas de verificación

Para casillas de verificación se utiliza el elemento <input> con type = "checkbox". Ejemplo:

<form>
  <p> Elija los ingredientes de su pizza: </p>
  <label for = "queso"> Queso extra </label>
  <input id = "queso" name = "ingrediente" type = "checkbox" value = "queso">
  <br>
  <label for = "pepperoni"> Pepperoni </label>
  <input id = "pepperoni" name = "ingrediente" type = "checkbox" value = "pepperoni">
  <br>
  <label for = "carne"> Carne </label>
  <input id = "carne" name = "ingrediente" type = "checkbox" value = "carne">
</form>

Salida:

En el ejemplo proporcionado hay valores asignados al atributo de value de las casillas de verificación. Estos valores no son visibles en el formulario en sí, por eso es importante que usemos una <label> asociada para identificar la casilla de verificación.
Cada <input> tiene el mismo valor para el atributo de name. El uso del mismo nombre para cada casilla de verificación agrupa los <input> 's. Sin embargo, cada <input> tiene una identificación única que se asocia con una <label>.

Botones de opción

Las casillas de verificación funcionan bien cuando se desea presentar a los usuarios varias opciones y permitirles elegir una o más de las opciones. Sin embargo, hay casos en los que se presentan varias opciones y solo se quiere permitir una selección, como preguntar a los usuarios si están de acuerdo o en desacuerdo con los términos y condiciones. Ejemplo:

<form>
  <p> ¿Cuánto es 1 + 1? </p>
  <input type = "radio" id = "dos" name = "respuesta" value = "2">
  <label for = "dos"> 2 </label>
  <br>
  <input type = "radio" id = "once" name = "respuesta" value = "11">
  <label for = "once"> 11 </label>
</form>


Salida:

Observe en el fragmento de código que los botones de opción (al igual que las casillas de verificación) no muestran su valor. Tenemos una <etiqueta> asociada para representar el valor del botón de radio. Para agrupar botones de radio, les asignamos el mismo nombre y solo se puede seleccionar un botón de radio de ese grupo.

Lista desplegable

Cuando se tiene una lista completa de opciones, la alternativa es utilizar una lista desplegable para permitir que los usuarios elijan una opción de una lista organizada. Ejemplo:

<form>
  <label for = "comida"> ¿Qué hay para comer? </label>
  <select id = "comida" name = "comida">
    <option value = "pizza"> Pizza </option>
    <option value = "sandwich"> Sandwich </option>
    <option value = "ensalada"> Ensalada </option>
    <option value = "ramen"> Ramen </option>
    <option value = "tacos"> Tacos </option>
  </select>
</form>

Salida:

En el código se está usando el elemento <select> para crear la lista desplegable.

Para completar la lista desplegable, se agregan varios elementos <option>, cada uno con un atributo value. De forma predeterminada, solo se puede seleccionar una de estas opciones. Cuando se envíe el formulario, la información de este campo de entrada se enviará utilizando el nombre del <select> y el valor de <option> elegida. Por ejemplo, si un usuario selecciona Pizza de la lista desplegable, la información se enviará como "comida = pizza".

Lista de datos

Cuando una lista desplegable tiene muchas opciones, puede ser tedioso para los usuarios desplazarse por toda la lista para localizar una opción. Para estos casos el elemento <datalist> resulta útil.

El elemento <datalist> se usa junto con un elemento <input type = "text">. <input> crea un campo de texto en el que los usuarios pueden escribir y filtrar opciones del <datalist>. Ejemplo:

<form>
  <label for = "ciudad"> ¿Ciudad ideal para visitar? </label>
  <input type = "text" list = "ciudades" id = "ciudad" name = "ciudad">

  <datalist id = "ciudades">
    <option value = "Nueva York"> </option>
    <option value = "Tokio"> </option>
    <option value = "Barcelona"> </option>
    <option value = "Ciudad de México"> </option>
    <option value = "Paris"> </option>
    <option value = "Otro"> </option>
  </datalist>
</form>
Observe que  <input> tiene un atributo list, cuyo valor está asociado al <datalist> a través del atributo id.

Salida:

Al hacer clic en el campo de entrada se muestra una lista desplegable.

Aunque <select> y <datalist> comparten algunas similitudes, existen algunas diferencias importantes. En el elemento <input> asociado, los usuarios pueden escribir en el campo de entrada para buscar una opción en particular. Si ninguna de las <option> coincide, el usuario aún puede usar lo que escribió. Cuando se envía el formulario, el valor de name de <input> y el valor de la opción seleccionada, o lo que el usuario ingresó , se envía como un par.

Área de texto

Un elemento <input> con type = "text" crea un campo de entrada de un solo renglón para que los usuarios ingresen información. Pero hay casos en los que los usuarios necesitan escribir más, como un comentario. Para estas situaciones se utiliza <textarea>.

El elemento <textarea> crea campos de texto más grandes. Se puede hacer uso de los atributos rows y cols para determinar el tamaño del campo. Ejemplo:

<form>
  <label for = "blog"> Nueva publicación de blog: </label>
  <hr>
  <textarea id = "blog" name = "blog" rows = "5" cols = "30">
  </textarea>
</form>

Salida:


Al enviar el formulario, el valor de <textarea> es el texto escrito dentro del cuadro.

Si se desea agregar un valor predeterminado al texto de <textarea> se debe hacer de la siguiente manera:

<textarea> Texto predeterminado </textarea>

Envío de formulario

El propósito de un formulario es recopilar información que se enviará para su procesamiento. Esa es la función del botón de envío: los usuarios hacen clic en él cuando terminan de completar la información del formulario. 

Para crear un botón de envío en un formulario se usa el elemento <input> con el atributo type="submit", y en el atributo value se coloca el texto que aparecerá en el botón. Por ejemplo:

<form>
<input type = "submit" value = "Enviar">
</form>

Salida:

Si no se utiliza el atributo value, el texto del botón será el configurado por default según el navegador utilizado.
